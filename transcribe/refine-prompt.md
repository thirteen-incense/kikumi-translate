# ASMR 中文字幕优化 Prompt（日文时间戳基准版）

用于 LLM 后处理阶段：以日文 LRC 的时间轴为基准，参考中文直译内容对中文字幕进行润色优化。

## ⚠️ 最重要：输出格式严格规定 — JSON 格式

**你必须严格按照以下 JSON 格式输出，行数必须与输入日文 LRC 完全一致：**

```json
{
  "lines": [
    {"idx": 1, "text": "翻译后的中文文本"},
    {"idx": 2, "text": "翻译后的中文文本"},
    {"idx": 3, "text": "翻译后的中文文本"}
  ]
}
```

- **行数：`lines` 数组长度必须与输入的日文 LRC 完全一致**。输入有多少行，你就要输出多少行。
- **idx：必须与输入的日文 LRC 中的序号完全一致**。不能改序号，不能跳号，不能重复。
- **text：每行的中文翻译文本**。未被判定为重复/幻觉的每一行都必须有非空文本。
- **禁止：Markdown 代码块、反引号、任何额外文字、空行、说明。**
- **禁止 JSON 中有任何多余字段。**

验证方法：输出后请自己数一下数组长度，确保与输入行数一致。

**只输出纯 JSON 对象，不得包含任何额外文字。**

## 铁律

> ⚠️ **输出必须仅为 JSON 对象本身**，不得包含：
> - ❌ "翻译完成"、"以上是优化结果"等总结
> - ❌ Markdown 代码块包裹
> - ❌ 任何说明、注释、工作汇报
> - ❌ 多余的空行首尾
> - ✅ 只输出 JSON，第一行就是 `{`，最后一行就是 `}`

## ⚡ 核心指令（优先级最高）

**输出行数必须与日文 LRC 完全一致。**
- 日文 LRC 有多少行，输出就有多少行
- 每行的**序号必须与日文 LRC 对应行的序号相同**，不可改变
- 对日文 LRC 的每一行：翻译日文→中文（**注意：抱く/抱いて/抱かれる/抱きたい/抱き方 = 干/做爱/上，绝不是「抱」**），再参考中文直译修正润色。
  利用该行**前后相邻行的日文内容**作为上下文理解语义，确保连贯。
- 中文直译 LRC 仅供参考语义，**不使用**它的行结构
- **最关键：日语的「抱く」（だく）及其所有活用形（抱いて、抱かれる、抱きたい、抱き方等）是性行为委婉语，必须翻译为「干/做爱/上」，绝不能翻译成字面「抱」。这是最高频错误，请反复核对。**

## 输入

两份以序号标记的 LRC：

- **日文 LRC** — Whisper large-v3 日语 ASR 输出的日文文本（**行数、行序以此为准，序号保持不动**）
- **中文直译 LRC** — Whisper ja_zh 模型日中直出的中文翻译（**仅供参考语义，不使用其行结构**）

可选附加信息：
- **作品元数据** — 标签（tags）、声优（VAs）、社团（circle），用于术语统一

## 输出

优化后的中文文本（**以日文序号为基准**）。
- 行数、行序 **以日文 LRC 为准**，日文 LRC 的每一条序号对应一行输出
- 对日文 LRC 的每一行：翻译日文→中文，再参考中文直译修正润色。
  利用该行**前后相邻行的日文内容**作为上下文理解语义，确保连贯。
- **格式：JSON，结构为 `{"lines": [{"idx": 序号, "text": "中文文本"}, ...]}`**（idx 与日文 LRC 完全一致）

## 优化规则

### 0. 全量校验（最高优先级）

⚠️ **这条规则最重要，优先级高于其他一切规则。**

Whisper ja_zh 模型的翻译错误率很高，且经常在连续多行上产生**内部自洽的虚构叙事**（幻觉级联），此时中文读起来完全通顺但内容与日文原文毫无关系。

**核心原则：三层语义基准（从高到低）**

| 优先级 | 语义基准 | 何时使用 |
|--------|---------|---------|
| ① 日文 LRC | 主要基准 | 日文识别正确时（大多数情况）|
| ② 上下文推理 | 兜底基准 | 日文 ASR 也出错时（同音错字、幻觉堆叠）|
| ③ 领域+情感基调 | 最终基准 | 中日文同时不可信时（双重幻觉）|

> **必须纠正的最高频错误示例：**
> 日文 `抱いて欲しい` → 正确：想被你干 / 想和你做爱 | 错误：想被你抱
> 日文 `抱かれて` → 正确：被干 / 被上 / 被抱被操 | 错误：被抱着
> 日文 `抱き方` → 正确：做爱的方式 / 干女人的方式 | 错误：抱的方式

### 1. 术语统一

参考作品 tags / VAs / circle 信息，确保以下 ASMR 专有名词翻译准确一致：

| 日文 | 推荐中文 | 避免 |
|------|---------|------|
| 耳舐め | 舔耳 | 耳朵被舔 |
| 抱く・抱いて・抱かれる・抱きたい（だく） | 干 / 做爱 / 上 | 抱（字面义）|
| おちんぽ | 肉棒 | 小鸡鸡 / 阴茎 |
| フェラ | 口交 | 口活 / 吹箫 |
| 手コキ | 手交 | 手淫 |
| クンニ | 舔阴 | 口交（混淆）|
| イく | 去了 / 高潮了 | 到了 |
| 気持ちいい | 好舒服 | 心情好 |
| 舐める | 舔 | 尝 / 吮 |
| 吸う | 吸 / 吮吸 | 吸允 |
| 挿入 | 插入 | 放进 |
| クリトリス | 阴蒂 | 豆豆 |
| おマンコ | 小穴 / 蜜穴 | 阴道 |
| 寝取らせ | 让别人睡我 / 绿帽Play | 放牛（致命误识） |

### 2. ja_zh 模型高发幻觉模式

> ⚠️ **特别警告：日语的「抱く」（だく）及其所有活用形（抱いて、抱かれる、抱きたい、抱き方等）是性行为委婉语，必须翻译为「干/做爱/上」，绝对不能翻成字面的「抱」。**

#### 2a. 高危：核心概念谐音误识
- 「寝とらせ」→ "放牛" → 修正为"让别人睡我"
- 「原坊/げんぼう」→ "愿望" → 修正为保留"原坊"
- 「射精/しゃせい」→ "写生" → 修正为"射精"

#### 2b. 高危：否定句反转
日文否定表达变成中文肯定。检查含「ない」「ぬ」「ません」的日文句，确保中文也表达否定。

#### 2c. 高危：幻觉级联
连续 3 行以上中文通顺但与日文无关 — 必须逐行按日文重写。

### 3. 否定句保护

| 日文句式 | 正确中文 | 错误直译 |
|---------|---------|---------|
| 〜わけないだろ | 不可能……吧 | 怎么可能不（反转）|
| 〜てはいけない | 不能…… | 可以…… |
| 〜んじゃない | 不是…… | 就是…… |

### 4. 句式优化

- **短句优先**：单句不超过 25 字
- **口语化**：使用"呢、吧、啊、呀、嘛"等语气词
- **拟声词保留**：ドクドク→噗通噗通、ビクビク→一抽一抽
- **角色名/人名**保留原文或音译，禁止翻译为中文实词

### 5. 行数保护 + 重复/幻觉处理

**基本原则**：
- **序号数值严格保持不变**（以日文 LRC 为准）
- **行数必须完全一致**，不得多一行也不得少一行

**重复行处理**：
- 检查相邻行：如果连续 2 行以上的文本内容高度相似（编辑距离 < 20%），判定为重复行
- 对于重复行：保留第 1 行的文本，后续重复行的文本置为空（该行输出空内容即可）

**幻觉行处理（优先级高）**：
- **末尾堆叠幻觉**：文件最后 3-5 行，如果中日文内容均为无意义的重复词（如"不同不同不同"、"好吗好吗"、"杀杀杀"、"010から"、"adder adder"等），判定为末尾幻觉
  - 处理方式：将这些行的文本置为空
- **单行插入幻觉**：某行中文的文本在日文 LRC 中对应区域明显不匹配（语义偏差 > 5 秒），且中文内容为通用填充词（"好吗"、"好吧"、"是啊"等），判定为插入幻觉
  - 处理方式：参考日文 LRC 对应区域的内容重写该行

**不得有空内容行** — 未被判定为重复/幻觉的每一行都必须有非空文本内容。
- **不得添加新行、不得删除现有行** — 行数严格与日文 LRC 保持一致

### 6. 输出格式

严格遵循 JSON 格式输出：

```json
{"lines": [{"idx": 1, "text": "字幕文本"}, {"idx": 2, "text": "字幕文本"}]}
```
